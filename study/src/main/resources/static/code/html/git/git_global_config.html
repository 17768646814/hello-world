配置之前可以使用测试命令检查与Git的连接情况

Git连接测试命令： 

$ ssh git@github.com

如果连接成功，可免去配置Git步骤

生成Git SSH Key

首先配置Git的username和email 

$ git config --global user.name "17768646814"
$ git config --global user.email "651158394@qq.com"
然后生成SSH密钥 
$ ssh-keygen -t rsa -C "651158394@qq.com"
此时会在~（用户目录，Windows下一般为C:/Users/域名）下生成.ssh文件夹。

ssh文件夹内主要包含“id_rsa”和“id_rsa.pub”文件，id_rsa是私钥，需要添加私钥到ssh-agent所管理的session中 

$ ssh-add ~/.ssh/id_rsa
如果报错：Could not open a connection to your authentication agent.，先执行 
$ eval `ssh-agent`
或者 
$ ssh-agent bash --login -i
（` => Tab上面那个键），在重新执行ssh-add命令 id_rsa.pub是公钥，将里面的内容添加至Git官网上，在执行Git连接测试命令。
